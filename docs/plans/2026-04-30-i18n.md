# Obsidian Gantt Calendar 插件 i18n 技术路线报告

> 日期：2026-04-30

## Context

当前插件所有用户可见字符串（~264 条）均以简体中文硬编码在 TypeScript 源码中，无任何 i18n 基础设施。插件面向中文用户（飞书集成、农历），但代码结构和 Obsidian 生态规范要求实现国际化，以便后续支持多语言和维护。

本报告综合了 Obsidian 官方文档、官方 API、社区插件工程实践三方面调研结果。

---

## 1. 官方 API 现状

### 语言检测 API（Obsidian 1.8+）

```typescript
// 静态方法
Platform.getLanguage()   // → "en" | "zh-cn" | "fr" | ...
// 插件实例方法
this.app.getLanguage()   // 同上
```

返回 ISO 语言代码，标准参考 [obsidianmd/obsidian-translations](https://github.com/obsidianmd/obsidian-translations)。

**兼容性方案（推荐）：**
```typescript
const locale = this.app.getLanguage?.()
  ?? window.moment?.locale?.()
  ?? "en";
```

当前 `manifest.json` 的 `minAppVersion` 为 `"1.5.0"`（早于 1.8），建议升级到 `"1.8.0"` 以直接使用 `getLanguage()`，同时保留 `moment.locale()` 回退以保证安全。

### 官方 ESLint 规则

[eslint-plugin-obsidianmd](https://github.com/obsidianmd/eslint-plugin) 提供了 `ui/sentence-case`、`ui/sentence-case-json`、`ui/sentence-case-locale-module` 规则，要求 UI 字符串使用句子大小写。可择机采纳。

### 官方不提供翻译框架

Obsidian 仅提供语言检测 API，不提供翻译字典加载、消息格式化、复数处理等框架。插件需自行实现或依赖第三方库。

## 2. 开源社区三种主流工程模式

### 模式 A：自定义 `localeMap` + `t()` 函数（社区共识推荐）

**代表插件：** obsidian-linter（10+ 语言）、admonitions（23 语言）、obsidian-leaflet（15 语言）

**结构：**
```
src/i18n/
  index.ts              # t() 函数 + initI18n()
  locales/
    en.ts               # 英文（源语言，通过 `as const` 推导所有键类型）
    zh-cn.ts            # 简体中文（Partial 类型，继承 en 键名）
```

**核心实现（~50 行）：**
```typescript
import en from "./locales/en";
import zhCN from "./locales/zh-cn";

type Translations = typeof en;
const localeMap: Record<string, Partial<Translations>> = { en, "zh-cn": zhCN };

let current: Partial<Translations> = localeMap.en;

export function initI18n(lang?: string) {
  const locale = lang ?? (window as any).app?.getLanguage?.() ?? "en";
  current = localeMap[locale] ?? localeMap.en;
}

export function t(key: keyof Translations, ...args: (string | number)[]): string {
  let str = (current[key] ?? en[key] ?? String(key)) as string;
  args.forEach((v, i) => { str = str.replace(`%${i + 1}`, String(v)); });
  return str;
}
```

**优点：** 零依赖，TypeScript 编译期键名校验，IDE 自动补全，社区广泛验证。
**缺点：** 无复数/上下文等高级功能，无动态加载。

### 模式 B：自捆绑 `i18next`

**代表插件：** obsidian-terminal（40+ 语言）、obsidian-vitepress（中英双语）、Enveloppe

**结构：**
```
src/i18n/
  i18next.ts            # i18next.init() 配置
  locales/
    en.json
    zh-cn.json
```

**优点：** 完整复数/插值/命名空间支持，JSON 格式便于社区贡献。
**缺点：** ~30KB 打包体积增加，配置复杂度高于自定义方案。

### 模式 C：复用 Obsidian 内部 `window.i18next`

直接访问 Obsidian 自身的 i18next 实例，通过 `addResourceBundle()` 注入翻译。

**优点：** 零体积开销。
**缺点：** 非官方 API，可能随版本变化而失效；命名空间冲突风险；不被 Obsidian 团队推荐。**不推荐使用。**

## 3. 社区 i18n 工具库

| 库 | 源 | 特点 |
|---|---|---|
| **obsidian-dev-utils** | [mnaoumov/obsidian-dev-utils](https://github.com/mnaoumov/obsidian-dev-utils) | 类型安全的 `createI18n()`，使用 `getLanguage()` API，Task-Board 等插件在用 |
| **@polyipseity/obsidian-plugin-library** | [polyipseity/obsidian-plugin-library](https://github.com/polyipseity/obsidian-plugin-library) | `mergeResources()` + `syncLocale()`，支持命名空间和交叉引用 `$t()` |
| **obsidian-i18n** | [eondrcode/obsidian-i18n](https://github.com/eondrcode/obsidian-i18n) | 面向终端用户翻译其他插件的工具（非开发库） |

## 4. 推荐技术方案

### 总路线：模式 A（自定义 `t()` 函数）

理由：
- 当前 ~264 条字符串规模，不需要 i18next 的高级功能
- 零额外依赖，不影响打包体积
- TypeScript 原生类型安全，无需额外工具链
- Obsidian 社区绝大多数中大型插件的选择
- 未来如需升级到 i18next，数据结构可平滑迁移

### 实施步骤

#### Phase 1：基础设施搭建（新增 3 个文件）

1. **`src/i18n/locales/en.ts`** — 英文源字典（`as const` 导出，作为键名类型来源）
2. **`src/i18n/locales/zh-cn.ts`** — 简体中文字典（`Record<TranslationKeys, string>` 确保键名完整）
3. **`src/i18n/index.ts`** — `initI18n()` 语言检测 + `t()` 翻译函数 + 类型导出

#### Phase 2：接入插件生命周期（修改 1 个文件）

4. **`src/main.ts`** — 在 `onload()` 最顶部调用 `initI18n()`，更新 `minAppVersion` 至 `"1.8.0"`

#### Phase 3：逐模块替换硬编码字符串（修改 ~20 个文件）

按用户可见优先级分批替换：
- **批次 1（高优先级）：** Settings builders（`src/settings/builders/*.ts`，~120 处）— `setName()` / `setDesc()` / `addDropdown()` 等
- **批次 2（高优先级）：** ContextMenu（`src/contextMenu/contextMenuIndex.ts`）和 Toolbar（`src/toolbar/*.ts`）
- **批次 3（中优先级）：** Modals（`src/modals/*.ts`）和 Notice 消息（`new Notice(...)` 全局替换）
- **批次 4（中优先级）：** Sidebar 视图（`src/sidebar/*.ts`）
- **批次 5（低优先级）：** 农历/节日数据 — 这些是中文文化固有内容，可保留原文或标记为 culture-specific

### 键名规范

采用 `模块.功能.描述` 的分层命名：
```
settings.sync.title          → "飞书任务同步"
settings.sync.button.fetch   → "获取任务清单"
notice.task.created          → "任务创建成功"
menu.edit                    → "编辑任务"
sidebar.search.placeholder   → "搜索任务..."
modal.create.title           → "创建新任务"
```

## 5. 验证方案

1. **类型检查：** `npm run build`（含 tsc 类型检查）确保所有 `t()` 调用键名有效，`zh-cn.ts` 覆盖全部键
2. **功能验证：**
   - 英语环境启动 Obsidian → 所有 UI 显示英文
   - 中文环境启动 → 所有 UI 显示中文
   - 切换语言后重启 → 生效
3. **完整性检查脚本：** 编写脚本扫描 `t("...")` 调用，对比 `locales/en.ts` 确保无遗漏
4. **回退测试：** 删除 `zh-cn.ts` 中随机键 → 确认回退到英文显示
