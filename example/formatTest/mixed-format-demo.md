# 混合格式演示

本文档演示了 Tasks 格式和 Dataview 格式混合使用的情况。

## 格式检测原理

插件会自动检测任务使用的格式：
- **Tasks 格式**：检测到优先级 emoji（🔺⏫🔼🔽⏬）或日期 emoji（➕🛫⏳📅✅❌）
- **Dataview 格式**：检测到字段格式 `[key:: value]`
- **混合格式**：同时包含两种格式的标记

## 混合格式示例

### 优先级混合

- [ ] 使用 emoji 优先级和字段日期 🔺 [created:: 2024-01-10] [due:: 2024-01-15]
- [ ] 使用字段优先级和 emoji 日期 [priority:: high] ➕ 2024-01-10 📅 2024-01-15

### 日期混合

- [ ] emoji 创建日期 + 字段到期日期 ➕ 2024-01-10 [due:: 2024-01-15]
- [ ] 字段创建日期 + emoji 到期日期 [created:: 2024-01-10] 📅 2024-01-15

### 完整混合

- [ ] 完整混合任务 ⏫ [created:: 2024-01-10] 🛫 2024-01-12 [scheduled:: 2024-01-15] 📅 2024-01-20
- [ ] 字段为主 + emoji 辅助 [priority:: high] [created:: 2024-01-10] 📅 2024-01-20 #work

## 格式使用建议

### 选择主要格式

建议在同一个笔记中主要使用一种格式，以保持一致性：

#### 主要使用 Tasks 格式
- [ ] Task 1 ⏫ ➕ 2024-01-10 📅 2024-01-15
- [ ] Task 2 🔼 ➕ 2024-01-12 📅 2024-01-20

#### 主要使用 Dataview 格式
- [ ] Task 1 [priority:: high] [created:: 2024-01-10] [due:: 2024-01-15]
- [ ] Task 2 [priority:: medium] [created:: 2024-01-12] [due:: 2024-01-20]

### 混合适用的场景

1. **过渡期**：从一种格式迁移到另一种格式时
2. **协作场景**：团队成员使用不同的格式
3. **特殊需求**：某些任务需要特殊的自定义字段

### 格式转换示例

#### Tasks 转 Dataview
```markdown
- [ ] 任务 ⏫ ➕ 2024-01-10 📅 2024-01-15
```
转换为：
```markdown
- [ ] 任务 [priority:: high] [created:: 2024-01-10] [due:: 2024-01-15]
```

#### Dataview 转 Tasks
```markdown
- [ ] 任务 [priority:: high] [created:: 2024-01-10] [due:: 2024-01-15]
```
转换为：
```markdown
- [ ] 任务 ⏫ ➕ 2024-01-10 📅 2024-01-15
```

## 兼容性说明

本插件完全兼容：
- **Tasks 插件**的 emoji 格式
- **Dataview 插件**的字段格式
- 两种格式的混合使用

无论使用哪种格式，插件都能正确解析和显示任务信息。
